# Security Policy

## Supported Versions

Security updates are provided for:

- The latest commit on the `main` branch
- The latest tagged release (when releases are published)

Older snapshots may not receive fixes.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use one of these channels:

1. Preferred: GitHub Security Advisories private reporting for this repository
   (`Security` tab -> `Advisories` -> `Report a vulnerability`).
2. Fallback: Contact the maintainer privately via GitHub profile:
   https://github.com/vessy

Include:

- A clear description of the issue and impact
- Steps to reproduce or a proof of concept
- Affected versions/commits
- Any suggested remediation (if available)

## Disclosure Process

- We will acknowledge receipt as soon as possible.
- We will validate and triage the report.
- We will work on a fix and coordinate disclosure timing.
- We may request a temporary embargo until a fix is available.

## Scope Notes

Security reports may include code in this repository and directly bundled
components under `Vendor/` where applicable.
